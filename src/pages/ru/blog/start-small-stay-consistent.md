Сейчас блог работает по очень простой схеме:

1. пишу пост в Markdown
2. пушу в `main`
3. GitHub Pages публикует автоматически

<figure>
  <img src="/images/blog-publishing-flow.svg" alt="Схема публикации от текста до живого сайта" />
  <figcaption>Минимум движущихся частей, максимум предсказуемости.</figcaption>
</figure>

Я выбрал этот путь, потому что он сохраняет темп:

- без привязки к CMS
- без серверов в обслуживании
- без скрытой сложности

Главное правило этого блога простое: сначала польза, потом полировка.
